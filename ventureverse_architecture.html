01
Input Reception

Next.js POSTs a structured JSON payload to POST /api/v1/analyze. FastAPI validates using Pydantic v2 models with custom validators for sector enums, numeric ranges, and date fields.

02
Feature Engineering

Categorical features are one-hot encoded. Free-text product description is embedded via OpenAI text-embedding-3-small. Derived metrics (runway months, burn multiple, team diversity score) are computed in NumPy.

03
ML Prediction Ensemble

XGBoost outputs a success probability. LightGBM classifies the most likely failure mode (product-market fit, cash flow, team, market timing). Random Forest produces a composite risk score. SHAP values are extracted per feature for explainability.

04
Vector Similarity Search

The startup's feature vector is embedded and queried against Qdrant's startup-profiles collection (cosine similarity, top-20 results). Results are filtered by sector and funding stage, then ranked by similarity score.

05
Graph Traversal

Neo4j Cypher queries identify common investor networks, failed-startup subgraph patterns, and founder background nodes that correlate with outcomes.

06
AI Agent Boardroom

Four specialized LLM agents receive the full analysis context. Each produces a domain-specific critique. The orchestrator synthesizes them into a unified board report using a structured prompt chain.

07
Recommendation Generation

RAG retrieves the 5 most similar historical success pivots from Qdrant. An LLM drafts a ranked, actionable recommendation plan. Redis caches the full response for repeat views.

FastAPI Main Service

  • Async request handling (uvicorn)
  • JWT auth middleware
  • Request routing → ML / Agent / Data services
  • WebSocket support for streaming Boardroom
  • Background task dispatch to Celery

Celery Workers

  • Heavy ML inference jobs
  • Dataset retraining pipelines
  • Batch embedding creation
  • Email/notification dispatch
  • Market data refresh (via Celery Beat)

PostgreSQL (RDS)

  • Multi-AZ deployment for HA
  • Read replicas for analytics queries
  • pgcrypto for sensitive field encryption
  • Alembic for schema migrations

Qdrant Vector DB

  • Collections: startup-profiles, graveyard, market-reports
  • 1536-dim OpenAI embeddings
  • HNSW index for sub-10ms ANN search
  • Payload filtering (sector, stage, geography)

Neo4j Graph DB

  • Nodes: Startup, Founder, Investor, Market
  • Relationships: INVESTED_IN, FOUNDED_BY, FAILED_DUE_TO
  • Pattern matching for failure subgraphs
  • Community detection (Louvain algorithm)

Redis (ElastiCache)

  • Analysis result cache (TTL 1h)
  • Session store for JWT refresh tokens
  • Celery message broker
  • Rate-limit counters per user

ML Models Used

  • XGBoost — Binary success classifier (AUROC target 0.87+)
  • LightGBM — Multi-class failure mode (8 categories)
  • Random Forest — Risk score regressor (calibrated 0–100)
  • Models retrained monthly on new data via Celery Beat + S3 model registry

LLM Routing Strategy

  • OpenAI GPT-4o → Investor Agent, Financial Agent (structured output)
  • Claude 3.5 Sonnet → Product Agent, Recommendation Writer (long-form)
  • Gemini 1.5 Pro → Marketing Agent, Market Pulse (web-grounded)
  • Fallback chain: primary → secondary → cached last response

Explainability (XAI)

  • SHAP TreeExplainer for tree models (fast, exact)
  • Top-10 feature importances per prediction
  • LLM translates SHAP output to plain English
  • Confidence intervals via bootstrap sampling

RAG Pipeline — Ingestion → Retrieval → Generation

flowchart TD subgraph SOURCES["Data Sources"] CB[Crunchbase Export\nStartup profiles · Funding rounds] CBI[CB Insights Reports\nMarket maps · Failure analysis] IND[Indian Startup Ecosystem\nInc42 · Tracxn data] FUND[Global Funding Datasets\nPitchBook · AngelList] PM[Post-Mortems\nStartup Lessons · Failory articles] end subgraph INGEST["Ingestion Pipeline"] PARSE[Data Parsers\nCSV · JSON · PDF extractors] CLEAN[Data Cleaner\nDedup · Normalize · Validate] CHUNK[Chunking Strategy\n512-token semantic chunks\nwith 64-token overlap] end subgraph EMBED["Embedding Pipeline"] EMB_SVC[OpenAI text-embedding-3-small\n1536 dimensions] META[Metadata Enrichment\nsector · stage · year · geography · outcome] end subgraph STORE["Vector Store — Qdrant"] COL1[Collection: startup-profiles\n~500K documents] COL2[Collection: failure-postmortems\n~50K documents] COL3[Collection: market-reports\n~20K documents] COL4[Collection: funding-rounds\n~1M documents] end subgraph RETRIEVAL["Retrieval Engine"] QUERY[Query Embedding\nUser startup vector] FILTER[Payload Filter\nsector + stage + geo] ANN[ANN Search\nHNSW cosine · top-20] RERANK[Cross-Encoder Reranker\ntop-20 → top-5] end subgraph GEN["Generation — RAG LLM Call"] PROMPT[Prompt Assembly\nSystem + Context chunks + Query] LLM_CALL[LLM Call\nClaude 3.5 Sonnet] GUARD[Output Guardrails\nHallucination check · Citation verify] RESP[Grounded Response\nwith source citations] end CB & CBI & IND & FUND & PM --> PARSE PARSE --> CLEAN --> CHUNK CHUNK --> EMB_SVC --> META META --> COL1 & COL2 & COL3 & COL4 QUERY --> FILTER --> ANN --> RERANK RERANK --> PROMPT COL1 & COL2 & COL3 & COL4 --> ANN PROMPT --> LLM_CALL --> GUARD --> RESP

Qdrant Collections

  • startup-profiles — Full startup records with outcome labels
  • failure-postmortems — Narrative failure analyses
  • market-reports — Sector & macro trend summaries
  • funding-rounds — All known investment events

Chunking Strategy

  • Structured profiles: one chunk per startup (dense embedding)
  • Post-mortems: semantic sentence-level chunking (512 tokens)
  • Market reports: paragraph-level with section metadata
  • Overlap: 64 tokens for cross-boundary context

Retrieval Quality

  • Two-stage: ANN (speed) → Cross-Encoder reranker (precision)
  • Payload filters prevent cross-sector noise
  • Reciprocal rank fusion merges multi-query results
  • Minimum relevance threshold: 0.72 cosine score

ECS Fargate Auto-Scaling

  • FastAPI: min 2, max 10 tasks — scale on CPU 60%
  • ML Service: min 1, max 4 — scale on queue depth
  • Celery Workers: min 2, max 8 — scale on SQS queue
  • Scheduled scale-down at 02:00 UTC

CI/CD Pipeline

  • GitHub Actions: lint → pytest → Docker build
  • Push to ECR on main branch merge
  • Blue/Green deployment via AWS CodeDeploy
  • Automatic rollback on health check failure
  • DB migrations run pre-deployment (Alembic)

Security

  • VPC with private subnets for all services
  • AWS Secrets Manager for all credentials
  • KMS encryption at rest for RDS + S3
  • IAM roles with least-privilege per service
  • WAF rules: SQLi, XSS, rate limiting

Cost Optimization

  • Fargate Spot for Celery workers (save 70%)
  • S3 Intelligent-Tiering for model artifacts
  • Redis cache reduces LLM API calls by ~60%
  • RDS reserved instances (1-year commitment)
  • LLM routing to cheapest model by task type
